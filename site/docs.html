Relayroom field manual

Run context handoffs without re-explaining the project.

Install Relayroom, point your agents at one handoff file, and capture the current state before you switch tools.

Install Setup Daily flow Adapters Commands Teams Safety

Step one

Install Relayroom

Install from the GitHub repo or run from a checkout. Relayroom is a Node CLI and requires Node 20 or newer.

npm install -g github:toyeshhm/relayroom
relayroom help

Step two

Initialize a project

Run these commands at the root of the repo you want agents to share. Relayroom creates `.relayroom/`, then installs small adapter blocks into each tool's native context file.

cd your-project
relayroom init
relayroom adapters install
What gets created

AGENTS.md for Codex, CLAUDE.md for Claude Code, .cursor/rules/relayroom.mdc for Cursor, and .relayroom/HANDOFF.md as the shared source of truth.

Core loop

Use one small habit before switching tools

01

Add durable context

Use notes for decisions, constraints, and facts the next agent should not rediscover.

relayroom note "Auth uses Supabase. Service role keys stay server-only."
02

Capture current state

Use capture before pausing, reviewing, or handing the repo to another tool.

relayroom capture \
  --task "Build checkout flow" \
  --note "Cart UI works; webhook tests remain."
03

Resume in the next agent

Print or copy an agent-specific prompt that includes the current handoff.

relayroom resume codex --print
relayroom resume claude --copy

How transfer works

Relayroom uses files the agents already read

There is no private API dependency. The adapters simply tell each tool to read `.relayroom/HANDOFF.md` before making changes.

Codex

AGENTS.md points Codex at the shared handoff.

Claude Code

CLAUDE.md points Claude Code at the same handoff.

Cursor

.cursor/rules/relayroom.mdc applies the handoff rule in Cursor.

Reference

Command reference

Command Use it for
relayroom init Create `.relayroom/`, config, notes, and the first handoff file.
relayroom adapters install Install Codex, Claude Code, and Cursor context pointers.
relayroom note "fact" Save durable project memory.
relayroom capture --task --note Snapshot the current work state into the handoff.
relayroom resume codex --print Print a ready-to-paste resume prompt for a target agent.
relayroom pack --agent cursor Export a standalone handoff pack.
relayroom status Inspect Relayroom and git state as JSON.

Working with others

Commit useful context, ignore private packs

Commit `.relayroom/HANDOFF.md`, `.relayroom/notes.md`, and adapter files when they help teammates. Relayroom ignores `.relayroom/packs/*.local.md` so personal exports do not leak into the repo by accident.

git add .relayroom AGENTS.md CLAUDE.md .cursor/rules/relayroom.mdc
git commit -m "Add Relayroom project handoff"

Guardrails

Do not store secrets in handoffs

Relayroom is plain text by design. It captures branch, status, diff stats, recent commits, and your notes. Keep API keys, tokens, customer data, and private credentials out of notes and captures.

Good handoff

OAuth callback is failing because EBAY_REDIRECT_URI does not match the app config.

Avoid

OAuth callback uses token abc123...
